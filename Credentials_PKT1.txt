1. Topology description:
----------------------------------------------------------------------------------------------------------------------

This topology is a simplified 3 tier architecture, which connects 4 separate zones (Floor1, Floor2, Floor3,Internet(host 10.0.0.86, for better testing) and Basement).
Each of the separate zone has its own ranges of IP addresses, VLANs, Access Policy, etc.


----------------------------------------------------------------------------------------------------------------------

 					(INTERNET)
 					    |

     |VLAN45||VLAN40||VLAN30|(Floor2) --- (CORE) --- (Basement | VLAN1 | Servers)

   				         /     \
   				  (Floor1)     (Floor3)
 				  |VLAN10|     |VLAN50|

Section: CORE
----------------------------------------------------------------------------------------------------------------------
=======================================================================================================================

This level is a Routing Core of a whole network and has no any VLANs, due to it uses IP routing. .
Simplified CORE Topology (_X and _Y) - Indexes of MLS:


            /----------------------------------------------------------------------------------------\
            |                                                                                        |
            |                                         (R_INT)                                        |
  	    |						 |                                           |
 	    |						 |                                           |
            |						 |                                           |
            |				/---------------(R1)----------------\                        |
 	    |				|	       /   \		    |			     |
 	    |				|	      /     \               |                        |
            | /--------------------(MLS_L3_2)        /       \            (R_B)-----(BASEMENT)       |
 	    | | 	     	     |	|           /  AREA0  \             |                        |
 	    | |		     	     |  |          /           \            |                        |
       /--(MLS_Y)           	     |	\-------(R2)----------(R3)----------/	                     |
       |        	     	     |		  \	      /                                      |
(FLOOR_ANY)     /--------------------/             \         /                                       |
       \---(MLS_X)                                  (MLS_L3_1)                                       |
             |                                       |     |                                         |
             \---------------------------------------/     \-----------------------------------------/
 
 
----------------------------------------------------------------------------------------------------------------------
NOTE:

The CORE topology uses Default SPINE-Leaf architecture, where MLS_L3_2 and MLS_L3_1 are Spine and Floor_ANY with its MLS_X and
MLS_Y are leafs,which connected to the Spines (MLS_L3_1 and MLS_L3_2). MLS_L3_1 and MLS_L3_2 is not directly connected.
Also each MLS connects to the 1 shared and 1 different router to provide better redundancy. The routers (R1,R2 and R3) are connected
between each other.

On the Core layer are no any ACL, Spanning-tree or broadcast, to provide fast performance of the network and routing process.
R_INT is connected directly to the R1. I decided to do it, due to R_INT was created only to demonstrate internet connection.
It has a weak point, so, it is recommended to connect it to the other routers.
