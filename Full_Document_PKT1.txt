1. Topology description:
----------------------------------------------------------------------------------------------------------------------

This topology is a simplified 3 tier architecture, which connects 4 separate zones (Floor1, Floor2, Floor3,Internet(host 10.0.0.86, for better testing) and Basement).
Each of the separate zone has its own ranges of IP addresses, VLANs, Access Policy, etc.


----------------------------------------------------------------------------------------------------------------------


				        |10.0.0.86|
					(INTERNET)
					    |
					    |
     |VLAN45||VLAN40||VLAN30|(Floor2) --- (CORE) --- (Basement | VLAN1 | Servers)
		         	          /   \
   				         /     \
   				  (Floor1)     (Floor3)
				  |VLAN10|     |VLAN50|
				  |VLAN20|     |VLAN60|

============================================================================================================================
WARNING: 

In this lab environment, SVIs are treated as trusted infrastructure interfaces.
Permitting all traffic sourced from SVI is a simplification to avoid over-complicating ACLs for control-plane and management protocols such as HSRP, DHCP relay, NTP, Syslog and SSH

----------------------------------------------------------------------------------------------------------------------------

HSRP control traffic is not affected by the applied VLAN ACLs due to its link-local multicast nature (TTL=1).
Additional permit statements for SVI-sourced traffic are included to avoid potential issues with management-plane traffic in Packet Tracer.

----------------------------------------------------------------------------------------------------------------------------

DNS access model :

End hosts are allowed to perform DNS resolution (UDP/TCP port 53) everywhere.
Direct access to DNS servers (ICMP, TCP other than 53) MAY be blocked by ACLs and policy requirements if needed.

For example, if the policy will be about denying access to the remote servers means that UDP/TCP 53 kinds of messages still 
won`t be discarded. 
----------------------------------------------------------------------------------------------------------------------------

Special Devices meaning:

Special device means device which IP address statically assigned and should be considered as trusted. Usually it is Layer 2 
Switches of Access and Distribution. They use VLAN default gateway as their own and belong to specific VLAN.
To simplify, it is the devices with SVI, but aren't used in routing the packets. It allows to provide
SSH, NTP, FTP, Syslog and other useful features even on Layer 2 devices

----------------------------------------------------------------------------------------------------------------------------
SVI and MLS meaning:

SVI means Switch Virtual Interface,which is assigned to Access, Distribution and Multilayer Switches. MLS means MultiLayer 
Switch
----------------------------------------------------------------------------------------------------------------------------
Banners:

Each device has its message of the day.
----------------------------------------------------------------------------------------------------------------------------
Remote and local server meaning:

Remote server means that server is located in the other network.
Local server means that server is located in the same network

VLAN10 
----------------------------------------------------------------------------------------------------------------------
1)VLAN10 uses the following IP range of addresses: 10.0.1.0/25, however addresses 10.0.1.122, .123, .124, .125 and .126 are reserved and cannot be assigned to the default clients. IP 10.0.1.122 is given to ASW3, .123 is given to DSW3. 10.0.1.124 — MLS4 and 10.0.1.125 — MLS3.

| VLAN | Subnet      | Gateway    | DHCP Range          | Reserved IPs |
| ---- | ----------- | ---------- | ------------------- | ------------ |
| 10   | 10.0.1.0/25 | 10.0.1.126 | 10.0.1.1-10.0.1.121 | 122–126      |


2)The VLAN10 uses ports only on ASW3. For clients Fa0/7-F0/24 including.
| Interface Type      | Ports                                                  |
| ------------------- | -------------------------------------------------------|
| **Trunk (Po1)**     | Fa0/1, Fa0/3                                           |
| **Trunk (Po2)**     | Fa0/2, Fa0/4                                           |
| **Access (VLAN10)** | Fa0/7-Fa0/9 (active), Fa0/10-24 (inactive)             |
| **Unused / VLAN1**  | Fa0/5(down), Fa0/6(down), Gi0/1(down), Gi0/2 (down)    |
| **SVI**             | Vlan10 (10.0.1.122), Vlan1 (down)                      |


3)VLAN10 default gateway: 10.0.1.126.

| Role            | Address           |
| --------------- | ----------------- |
| Virtual IP      | 10.0.1.126        |
| Active HSRP     | MLS3 - 10.0.1.125 |
| Standby HSRP    | MLS4 - 10.0.1.124 |
| HSRP Preemption | Enabled           |



4)VLAN10 Clients use local DHCP server(MLS3) and remote DNS server (host 10.0.0.226).

5)VLAN10 Clients are allowed to use internet(host 10.0.0.86) and DNS resolution (UDP/TCP 53), but are not allowed to directly access or reach the DNS server (ICMP, TCP other than 53 is denied)

6)VLAN10 Special devices (10.0.1.122 (asw3.floor1.com), 10.0.1.123 (dsw3.floor1.com), 10.0.1.124 (MLS4's SVI) and 10.0.1.125 (MLS3's SVI) ) can access services such as (T)FTP, SSH, Syslog, DNS and NTP. 

7)The root bridge for the VLAN10 is DSW3 and configured by lowering STP priority for this VLAN.

8)The extended ACL is applied directly on the VLAN's interface on the MLS3/4. 
The direction is IN.


Extended IP access list VLAN_10
    10 permit udp any any eq bootpc                                ! Permitting DHCP Server and Clients messages
    20 permit udp any any eq bootps 
    30 permit ip 10.0.1.0 0.0.0.127 host 10.0.0.86                 ! Permitting internet access, Internet is host 10.0.0.86
    40 permit tcp 10.0.1.0 0.0.0.127 host 10.0.0.226 eq domain     ! Permitting DNS TCP/UDP 53 messages
    50 permit udp 10.0.1.0 0.0.0.127 host 10.0.0.226 eq domain
    60 permit ip host 10.0.1.125 any                               ! permitting infrastructure-originated traffic 
    70 permit ip host 10.0.1.124 any 				   ! (management/control-plane simplification
    80 permit ip host 10.0.1.122 10.0.0.224 0.0.0.31               ! Permitting to ASW3 and DSW3 connectivity to the Basement 
    90 permit ip host 10.0.1.123 10.0.0.224 0.0.0.31 
    100 deny ip any any                                            !deny other traffic / to monitor how many packets were denied
